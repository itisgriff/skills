# Model Definitions with @Table

## The @Table Macro

Use the `@Table` macro from StructuredQueries to define models. It generates query helpers, column accessors, and a `Draft` type for inserts.

```swift
@Table
struct Item: Identifiable {
    let id: UUID
    var name: String
    var isComplete: Bool
}
```

Custom table name:

```swift
@Table("reminder_items")
struct Item: Identifiable {
    let id: UUID
    var name: String
}
```

## Swift 6 Compliance

Mark `@Table` structs as `nonisolated` for Swift 6 strict concurrency:

```swift
nonisolated @Table
struct Item: Identifiable {
    let id: UUID
    var name: String
}
```

## The @Column Macro

Use `@Column` to customize column names or specify generated columns:

```swift
@Table
struct Item {
    let id: UUID

    @Column("display_name")
    var name: String

    @Column(generated: .virtual)
    let computedField: String
}
```

## The @Selection Macro

Use `@Selection` to define custom projections — subsets of columns or computed values:

```swift
@Table
struct Player: Identifiable {
    let id: UUID
    var name: String
    var score: Int
    var isInjured: Bool

    @Selection
    struct Summary {
        let name: String
        let score: Int
    }
}
```

Use selections in queries:

```swift
@FetchAll(Player.all.select(Player.Summary.self)) var summaries: [Player.Summary]
```

## The Draft Type

`@Table` generates a `Draft` nested type used for inserts. `Draft` omits the primary key so the database can assign it:

```swift
// Generated by @Table:
// Item.Draft(name: String, isComplete: Bool)

try Item.insert { Item.Draft(name: "New Item", isComplete: false) }.execute(db)
```

For tables where you supply the primary key (e.g., UUID), include it in the `Draft`:

```swift
try Item.insert { Item(id: UUID(), name: "New Item", isComplete: false) }.execute(db)
```

## Primary Key Conventions

- Use `UUID` for CloudKit-synced tables (required for `IdentifierStringConvertible`)
- Use `let` for primary keys (immutable)
- Default UUID generation in SQL: `DEFAULT (uuid())`

## STRICT Tables

Always use `STRICT` mode when creating tables in migrations for type safety:

```swift
try #sql("""
    CREATE TABLE "items" (
        "id" TEXT PRIMARY KEY NOT NULL DEFAULT (uuid()),
        "name" TEXT NOT NULL,
        "isComplete" INTEGER NOT NULL DEFAULT 0
    ) STRICT
""").execute(db)
```

## Foreign Keys

Define foreign key relationships using column types that match the referenced table's primary key:

```swift
@Table
struct Attendee: Identifiable {
    let id: UUID
    var name: String
    var itemID: Item.ID  // Foreign key to Item
}
```

In the migration:

```swift
try #sql("""
    CREATE TABLE "attendees" (
        "id" TEXT PRIMARY KEY NOT NULL DEFAULT (uuid()),
        "name" TEXT NOT NULL,
        "itemID" TEXT NOT NULL REFERENCES "items"("id") ON DELETE CASCADE
    ) STRICT
""").execute(db)
```
